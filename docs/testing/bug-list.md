### P0 - 严重 (Critical)

#### BUG-001: 8080 端口被占用，后端服务无法启动

**发现日期**: 2026-03-10  
**发现者**: AI 测试工程师  
**状态**: 🔴 待修复  
**优先级**: P0  
**影响范围**: 后端 API 全部功能  

**问题描述**:
端口 8080 被 SearXNG 服务占用，导致 AI 学习平台后端无法启动。

**复现步骤**:
1. 检查端口占用：`netstat -tlnp | grep 8080`
2. 尝试启动后端服务：`go run cmd/main.go`
3. 服务启动失败或绑定到错误端口

**当前状态**:
```
tcp   0   0   127.0.0.1:8080   0.0.0.0:*   LISTEN   searxng
```

**影响**:
- 所有后端 API 无法访问
- 前端无法连接后端
- 无法进行 API 测试

**建议解决方案**:
1. **方案 A (推荐)**: 停止 SearXNG 容器
   ```bash
   docker stop searxng
   ```

2. **方案 B**: 修改后端服务端口
   - 编辑 `.env` 文件，设置 `PORT=8081`
   - 更新前端 API 配置

3. **方案 C**: 配置反向代理
   - 使用 Nginx 将 `/api/` 路径代理到后端端口

**关联任务**: 
- 需要运维协调
- 需要更新部署文档

#### BUG-002: Go 运行环境未安装

**发现日期**: 2026-03-10  
**发现者**: AI 测试工程师  
**状态**: 🔴 待修复  
**优先级**: P0  
**影响范围**: 后端服务部署、本地开发  

**问题描述**:
开发机器未安装 Go 1.21+，无法编译和运行后端服务。

**复现步骤**:
1. 执行 `go version`
2. 返回 `command not found`

**当前状态**:
```bash
$ which go
# 无输出
```

**影响**:
- 无法本地运行后端服务
- 无法进行代码修改和测试
- 依赖 Docker 容器运行

**建议解决方案**:
1. 安装 Go 1.21+:
   ```bash
   # 使用官方安装脚本
   wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz
   sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz
   export PATH=$PATH:/usr/local/go/bin
   ```

2. 或使用 Docker 运行:
   ```bash
   docker run --rm -v $(pwd):/app -w /app golang:1.21 go run cmd/main.go
   ```

**关联任务**:
- 更新 DEVELOPMENT.md 添加安装说明
- 考虑使用 Docker 统一开发环境
