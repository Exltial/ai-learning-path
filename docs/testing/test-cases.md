## 1. 认证模块测试用例

### 1.1 用户注册 (POST /api/v1/auth/register)

| 用例 ID | AUTH-REG-001 |
|---------|-------------|
| **标题** | 成功注册新用户 |
| **优先级** | P0 |
| **前置条件** | 数据库连接正常，用户名和邮箱未被注册 |
| **测试步骤** | 1. 发送 POST 请求到 /api/v1/auth/register<br>2. 请求体包含有效的 username, email, password<br>3. 验证响应 |
| **输入数据** | `{"username": "testuser", "email": "test@example.com", "password": "SecurePass123"}` |
| **预期结果** | - 状态码 201<br>- success: true<br>- 返回用户信息（不含密码）<br>- 返回 JWT token |
| **测试类型** | 单元测试、集成测试 |

| 用例 ID | AUTH-REG-002 |
|---------|-------------|
| **标题** | 注册时用户名已存在 |
| **优先级** | P1 |
| **前置条件** | 用户名 "testuser" 已存在于数据库 |
| **测试步骤** | 1. 发送注册请求，使用已存在的用户名 |
| **输入数据** | `{"username": "testuser", "email": "new@example.com", "password": "SecurePass123"}` |
| **预期结果** | - 状态码 409<br>- success: false<br>- error.code: "CONFLICT" |
